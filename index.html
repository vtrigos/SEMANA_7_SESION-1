CSS: Como lo hemos venido usando

Hasta ahora para crear estilos para cambiar el diseño de nuestro html usamos una hoja de estilo CSS (Lenguaje reconocido por el navegador pero no es un lenguaje de programación) que nos permite controlar el diseño y darle formato a nuestras páginas web como definir colores, fuentes, márgenes, alineación y distribución de los elementos de la pàgina web.

Para llamar a nuestro CSS en nuetro html hemos usado 3 formas:

  1. Uso externo: Enlazamos nuetros CSS mediante la etiqueta <link>, Ejemplo: <link rel="stylesheet" href="./style.css>
    que colocaremos dentro de nuestra etiqueta <head> y así llamaremos a todo el código que generaremos a un archivo CSS externo.
  2. Uso interno: Colocaremos CSS dentro de nuestra página HTML usando la etiqueta <style> dentro de <head>.

    Ejemplo:

    
        <head>
            <style>
                body {
                    background-color: blue;
                }
            </style>
        </head>
  3. Uso inline: Uso directo a los elementos HTML usando el atributo style.

    Ejemplo:

    <h1 style="color: blue;"> Título de muestra<h1>

DESVENTAJAS CSS PLANO

Hasta ahora hemos utilizado la forma del llamado externo pero esto es más recomendable para proyectos pequeños por ejemplo crear una landing page. A continuación indicamos algunas desventajas de usar CSS PLANO:

  • Si nuestro proyecto tiene muchas páginas web es muy probable que nuestro CSS tenga muchos bloques de código CSS (Ejm. más de 10000 líneas) y esto hace que la lectura o ubicación de elementos sea complicada.
  • Si quisieramos crear varios CSS para facilitar ubicación y lectura del code por ejemplo tendriamos que hacer la llamada en el html por ejemplo si crearamos 10 CSS tendriamos que hacer 10 llamadas en al web y no es recomendable para el tema de performance y posicionamiento.
  • No permite hacer escalable el CSS, no podemos automatizar y reutilizar.
  • El trabajo no se puede hacer modular y dificulta el trabajo en equipo.

QUE ES UN PROCESADOR

Un procesador CSS es una herramienta que escala las capacidades del CSS plano al permitir el uso de variables, anidación, funciones, operaciones matemáticas, etc. Esta herramientas procesa el código fuente escrito en un lenguaje de preprocesador y lo convierte en CSS estándar, que los navegadores puedan entender. Entre los más conocidos tenemoa a: STYLUS, LESS y SAS

VENTAJAS DEL PROCESADOR

  • Permite convertir un CSS plano a escalable.
  • Mejoran la productividad y mantenimiento del código CSS.
  • Nos permite estructurar CSS usando la modularidad (Componentes y secciones) para nuestros proyectos podemos tener varios archivos para un mejor manejo del CSS y evitar sobrecarga de code.
  • Reduce el tema de sobreescritura.
  • Permite usar valores reutilizables para facilitar el control de cambios en el diseño.
  • Administrar guìas de estilos mediante variables.
  • Optimiza y permite el trabajo de equipo en un proyecto grande.

QUÉ ES SASS

Es un lenguaje que extiende las capacidades del CSS estándar, usa arquitectura y nomenclatura CSS. SASS usa un compilador que permite generar un archivo css para que este sea integrado en el html y asì el navegador lo reconozca. Es decir todo lo que escribamos en SASS se traducira en CSS en el archivo generado automáticamente.

Al trabajar de forma modular SASS podemos crear varios archivos (parches) que serán importados en 1 principal y este es el que se compila.

Ejemplo: Puedo crear mi archivo styles.scss y este lo compilo (generará 2 archivos: styles.css y styles.css.map) cada vez que grabe genera/actualiza el css en el archivo styles.css por eso el compilado debe estar activo. Si tuvieramos varios parches debemos evitar el compilado pues sólo 1 se compila, entonces para evitar que un parche sea compilado debemos colocar al inicio del nombre del archivo parche el guión bajo ( ejm.: _fuentes.scss )

VARIABLES

Contenedor que permite guardar valores que se pueden reutilizar en todo nuestro proyecto CSS.

Ejm.: Usando variable $color-titulo-principal

 
        $color-titulo-principal: #3498db;
        $alto-linea: 1.5;
        
        h1 {
            color: $color-titulo-principal;
        }

        body {
            line-height: $alto-linea; 
        }

SCOPE

Se refiere al alcance o ámbito de una variable

Ejm.: Usando variable $color-subtitulos, en la clase color-sass variará su valor tras nueva asignación.

 
        $color-subtitulos: #cd0606;

        h2 {
            color: $color-subtitulos;
        } 
        
        .color-sass {
            $color-subtitulos: #0f5920;
            color: $color-subtitulos;
        }

ANIDAMIENTO

Nos permite escribir CSS de forma estructurada y con jerarquía aplicando estilos de manera más organizada y estableciendo un mejor entendimiento de la relación de padre e hijo entre selectores mediante el anidado (uno dentro de otro) pudiendo ser 2 o más anidados(para ramificaciones de parientes).

Ejm.: Aquí b es hijo de p por ende todas las negritas que se encuentren dentro de la etiqueta p serán de color azul.


        p {
            b {
                color: blue;
            }
        }

AMPERSAN PADRE

El uso de & en SASS se usa para referirse al selector padre dentro de un anidamiento de manera eficiente y mantener el código CSS más organizado y fácil de entender.

Ejm.: Aquí notamos como la pseudo clase se refiere directamente a la clase botón mediante &.

 
        .boton {
            background-color: #9927c2;
            color: white;
            padding: 10px 20px;
            text-decoration: none;
            border-radius: 4px;
            transition: background-color 0.3s ease;
          
            &:hover {
              background-color: #2941b9;
            }
          }


IMPORT

Se usa para dividir (modular) css en varios archivos (parches) pero como al final se compilará sólo uno debe anteponerse a los archivos parches el "_", estos serán llamados al archivo principal mediante @import

Ejm.: _maindata.scss

Este archivo parche será llamado mediante import en index.html : @import 'maindata';
